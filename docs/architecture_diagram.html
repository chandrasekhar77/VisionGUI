Vision Inspection Platform — 4-Layer Architecture

Layer 4 — UI / Management Layer
Dark theme · Frameless · Dockable panes · Lightweight UI thread
Shell (constant)
  • MainFrm — layout, module switching
  • TopBar — nav pills, Connect/Start/Stop
  • DarkPane — dockable panes
  • UITheme — colors, dimensions
Module (swappable)
  • InspectionModule
  • CalibrationModule
  • SetupModule
  • implements IVisionModule
Views (10 tabs)
  • Monitoring Dashboard
  • Real-time Results
  • Recipe / Config
  • Statistics & Defect Viewer
  • Camera / Hardware / Lighting
Contract — L3 ↔ L4
ISequenceController IStatusPublisher IResultPublisher IStatisticsPublisher PostMessage (worker → UI)
Layer 3 — Business Layer
Orchestrates HAL + Inspection · No vendor SDK calls · No UI widget calls
Sequence Controller
  • Grab → Detect Panel
  • Set lighting per ROI
  • Submit to Inspection
  • Collect result
  • Calls L1 HAL directly
  • Calls L2 Inspection directly
Pass / Fail Evaluation
  • Aggregate ROI results
  • Final panel verdict
  • Apply tolerance rules
Result Reporting
  • Upload to server
  • ITcpClient (JSON)
  • 4-byte length prefix
Contract — L2 ↔ L3
IInspectionEngine IInspectionResult IModelLoader IModel
Layer 2 — Inspection Layer
Pure image processing · No hardware calls · No UI calls
Image Processing
  • MIL
  • OpenCV
  • Custom Algorithms
  • Binarization, Morphology
Rule-Based Engine
  • MIL / OpenCV / Custom
  • Selected per ROI in model
  • Deterministic
Deep Learning Engine
  • Neural Network
  • GPU accelerated
  • Same model spec as Rule-Based
Model / Recipe System
  • Golden image
  • ROI definitions
  • Engine type per ROI
  • Tolerance / parameters
Contract — L1 ↔ L2/L3
ICamera IImageFrame IRingBuffer ILightController IIODevice IMotionController ITcpClient

Key Architecture Rules
Interface first — contract before implementation
No layer skipping — L4 never calls L1 directly
One direction of knowledge — lower layers know nothing above
Sequence Controller may call L1 HAL directly
L2 Inspection is pure — no hardware calls
Config drives vendor / engine selection
UI thread zero blocking — PostMessage only
Simulated HAL — full system without hardware
Golden dataset regression after every algorithm change
Never call vendor SDK outside HAL layer
Model is single source of truth — no magic numbers
Same EXE on all PCs — features enabled/disabled by role config
